什麼是 Flask？

Flask 是一個輕量級的 Python Web 框架，基於 Werkzeug 工具箱和 Jinja2 模板引擎。它被設計為易於上手且易於擴展，適合建構小型到中型的 Web 應用。

Flask 的主要特色：

  • 簡單易學： 极簡核心，易於理解和使用
  • 靈活性高： 不強制特定結構或依賴，開發者有高度自由度
  • 廣泛擴展： 豐富的擴展生態系統（Flask-Login、Flask-WTF、Flask-SQLAlchemy 等）
  • 內建開發伺服器： 快速測試和開發
  • Unicode 支援： 良好的國際化支援

為什麼選擇 Flask？

Flask 適用於以下場景：

  • 學習 Web 開發： 簡單直接，適合初學者理解 Web 基礎概念
  • 原型開發： 快速構建 MVP 和概念驗證
  • 小型應用： 個人網站、部落格、內部工具
  • 微服務： 作為輕量級 API 服務
  • 教育用途： 教學和演示 Web 開發概念

模板引擎 Jinja2

Flask 使用 Jinja2 作為模板引擎，允許在 HTML 中嵌入 Python 語法。

# app.py 中的路由 @app.route("/") def home(): user = "Porsche哥" posts = ["第一篇文章", "第二篇文章", "第三篇文章"] return render_template('index.html', user=user, posts=posts)
# templates/index.html <!DOCTYPE html> <html lang="zh-TW"> <head> <meta charset="UTF-8"> <title>{{ user }} 的網站</title> </head> <body> <h1>歡迎，{{ user }}！</h1> <h2>我的文章</h2> <ul> {% for post in posts %} <li>{{ post }}</li> {% endfor %} </ul> <!-- 條件判斷 --> {% if user == "Porsche哥" %} <p>你好，站長！</p> {% else %} <p>你好，訪客！</p> {% endif %} <!-- 包含其他模板 --> {% include 'header.html' %} </body> </html>

靜態文件

Flask 自動提供 /static 路徑來服務靜態文件（CSS、JavaScript、圖片等）。

# 目錄結構 yourproject/ ├── app.py ├── static/ │ ├── css/ │ │ └── style.css │ ├── js/ │ │ └── script.js │ └── images/ │ └── logo.png └── templates/ ├── index.html └── about.html
# 在模板中使用靜態文件 <link rel="stylesheet" href="{{ url_for('static', filename='css/style.css') }}"> <img src="{{ url_for('static', filename='images/logo.png') }}" alt="Logo"> <script src="{{ url_for('static', filename='js/script.js') }}">

表單處理與驗證

Flask 本身不包含表單處理，但常配合 Flask-WTF 使用。

# 安裝 Flask-WTF pip install flask-wtf
from flask import Flask, render_template, redirect, url_for, flash from flask_wtf import FlaskForm from wtforms import StringField, PasswordField, SubmitField from wtforms.validators import DataRequired, Length, EqualTo app = Flask(__name__) app.config['SECRET_KEY'] = 'your-secret-key-here' class LoginForm(FlaskForm): username = StringField('使用者名稱', validators=[DataRequired(), Length(min=4, max=25)]) password = PasswordField('密碼', validators=[DataRequired()]) submit = SubmitField('登入') @app.route("/login", methods=['GET', 'POST']) def login(): form = LoginForm() if form.validate_on_submit(): flash('登入成功！', 'success') return redirect(url_for('index')) return render_template('login.html', form=form)

資料庫整合

Flask 常用 SQLAlchemy 來處理資料庫操作。

# 安裝相關套件 pip install flask-sqlalchemy
from flask import Flask from flask_sqlalchemy import SQLAlchemy app = Flask(__name__) app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///site.db' app.config['SQLALCHEMY_TRACK_MODIFICATIONS'] = False db = SQLAlchemy(app) class User(db.Model): id = db.Column(db.Integer, primary_key=True) username = db.Column(db.String(20), unique=True, nullable=False) email = db.Column(db.String(120), unique=True, nullable=False) password = db.Column(db.String(60), nullable=False) def __repr__(self): return f"User('{self.username}', '{self.email}')"

RESTful API 開發

Flask 可以輕鬆建構 RESTful API。

from flask import Flask, jsonify, request from flask_sqlalchemy import SQLAlchemy app = Flask(__name__) app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///items.db' db = SQLAlchemy(app) class Item(db.Model): id = db.Column(db.Integer, primary_key=True) name = db.Column(db.String(80), nullable=False) description = db.Column(db.String(200)) # 建立資料表 with app.app_context(): db.create_all() # GET 所有項目 @app.route("/api/items", methods=['GET']) def get_items(): items = Item.query.all() return jsonify([{ 'id': item.id, 'name': item.name, 'description': item.description } for item in items]) # GET 單個項目 @app.route("/api/items/<int:item_id>", methods=['GET']) def get_item(item_id): item = Item.query.get_or_404(item_id) return jsonify({ 'id': item.id, 'name': item.name, 'description': item.description }) # POST 建立新項目 @app.route("/api/items", methods=['POST']) def create_item(): data = request.get_json() new_item = Item(name=data['name'], description=data.get('description', '')) db.session.add(new_item) db.session.commit() return jsonify({'id': new_item.id}), 201 # PUT 更新項目 @app.route("/api/items/<int:item_id>", methods=['PUT']) def update_item(item_id): item = Item.query.get_or_404(item_id) data = request.get_json() item.name = data.get('name', item.name) item.description = data.get('description', item.description) db.session.commit() return jsonify({'id': item.id}) # DELETE 刪除項目 @app.route("/api/items/<int:item_id>", methods=['DELETE']) def delete_item(item_id): item = Item.query.get_or_404(item_id) db.session.delete(item) db.session.commit() return '', 204

錯誤處理

from flask import Flask, render_template app = Flask(__name__) @app.errorhandler(404) def not_found_error(error): return render_template('404.html'), 404 @app.errorhandler(500) def internal_error(error): db.session.rollback() return render_template('500.html'), 500

應用工廠模式

對於較大的應用，建議使用應用工廠模式來提升可組織性和可測試性。

# project/__init__.py from flask import Flask from flask_sqlalchemy import SQLAlchemy db = SQLAlchemy() def create_app(): app = Flask(__name__) app.config.from_object('config.Config') db.init_app(app) # 註冊藍圖 from .routes import main_bp app.register_blueprint(main_bp) return app
# project/routes.py from flask import Blueprint, render_template main_bp = Blueprint('main', __name__) @main_bp.route("/") def index(): return render_template('index.html')
# run.py from project import create_app app = create_app() if __name__ == "__main__": app.run(debug=True)

部署與生產環境

生產環境部署建議：

  • 使用 Gunicorn 或 uWSGI 作為 WSGI 伺服器
  • 設定適當的工作進程數
  • 使用過程管理器如 Supervisor 或 systemd
  • 設定日誌輪轉和監控
  • 使用反向代理如 Nginx
  • 啟用適當的安全標頭（CSP、HSTS 等）
  • 設定環境變數來管理配置
# 使用 Gunicorn 部署 gunicorn -w 4 -b 0.0.0.0:8000 app:app

Flask vs FastAPI 比較

⚡ 效能比較

FastAPI 通常比 Flask 快 2-3 倍，特別是在異步操作和高併發場景下。

📚 學習曲線

Flask 更簡單直觀，適合初學者；FastAPI 需要理解 Python 類型提示和異步概念。

🔧 特色功能

Flask: 成熟生態系統、靈活性高、廣泛採用
FastAPI: 自動文件、類型安全、異步支援、依賴注入

🎯 使用場景建議

選擇 Flask 時：
- 需要最大靈活性和控制權
- 團隊對 Flask 已有經驗
- 建構傳統多頁面應用
- 需要廣泛的擴展選擇

選擇 FastAPI 時：
- 建構高效能 API 或微服務
- 需要自動生成的互動式文件
- 重視類型安全和開發工具支援
- 應用大量異步 I/O 操作

結語

Flask 作為 Python Web 框架的經典選擇，以其簡潔、靈活和成熟的生態系統贏得了廣泛的採用。無論是學習 Web 開發、建構個人專案，或是開發企業級應用，Flask 都提供了一個堅實的基礎。

雖然 FastAPI 在效能和現代特性上具有優勢，但 Flask 的簡單性和廣泛適用性使其在許多場景下仍然是優秀的選擇。選擇哪個框架應該基於具體的專案需求、團隊經驗和長期維護考量。

💡 學習建議：從基本的路由和模板開始練習，逐步探索表單處理、資料庫整合和API開發。實際專案練習是掌握 Flask 的最佳途徑。
