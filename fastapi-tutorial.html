什麼是 FastAPI？

FastAPI 是一個現代、快速（高效能）的 Python Web 框架，用於構建 API（應用程式介面），基於標準 Python 類型提示。

FastAPI 的主要特色：

  • 極高效能： 速度比肩 NodeJS 和 Go（歸功於 Starlette 和 Pydantic）
  • 快速開發： 功能開發速度提升 200% 到 300%
  • 少量錯誤： 透過類型提示減少約 40% 的人為錯誤
  • 內建文件： 自動生成互動式 API 文件（Swagger UI、ReDoc）
  • 標準基礎： 基於完全標準的 Python 類型提示

為什麼選擇 FastAPI？

在現代微服務和雲原生應用中，FastAPI 提供了以下優勢：

  • 異步支援： 原生支援 async/await，適合高併發場景
  • 類型安全： 基於 Pydantic 的資料驗證和序列化
  • 依賴注入： 強大而直觀的依賴注入系統
  • 安全性： 內建認證和授權工具
  • 測試友善： 基於 Starlette，測試非常簡單

# 安裝 FastAPI 和伺服器 pip install fastapi pip install uvicorn[standard]

# 運行應用 uvicorn main:app --reload

from fastapi import FastAPI, HTTPException from typing import Dict app = FastAPI() items = { "foo": {"name": "Foo Fighters"}, "bar": {"name": "Bar Bar"} } @app.get("/items/{item_id}") async def read_item(item_id: str): if item_id not in items: raise HTTPException(status_code=404, detail="Item not found") return items[item_id]

背景任務

from fastapi import FastAPI, BackgroundTasks from typing import Dict app = FastAPI() def write_notification(email: str, message: str = ""): with open("log.txt", "mode") as email_file: content = f"notification for {email}: {message}" email_file.write(content) @app.post("/send-notification/{email}") async def send_notification(email: str, background_tasks: BackgroundTasks): background_tasks.add_task(write_notification, email, message="Hello from FastAPI!") return {"message": "Notification sent in the background"}

部署與生產環境

生產環境部署建議：

  • 使用 Gunicorn 與 Uvicorn 工作器：gunicorn -k uvicorn.workers.UvicornWorker main:app
  • 設定適當的工作器數量：通常為 (2 x CPU核心數) + 1
  • 使用過程管理器如 Supervisor 或 systemd
  • 設定日誌輪轉和監控
  • 使用反向代理如 Nginx

結語

FastAPI 代表了現代 Python Web 框架的發展方向，結合了高效能、開發效率和現代特性。無論是建構微服務、建立內部工具，或是創建公開的 API，FastAPI 都是一個優秀的選擇。

隨著其生態系統的成長和社群的擴大，FastAPI 正在成為構建高品質 Python 應用的標準選擇之一。
