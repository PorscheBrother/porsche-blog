第一部分：Docker 基礎入門

什麼是 Docker？

Docker 是一個容器化平台，讓應用程式可以在隔離的環境中運行。想像成「輕量級虛擬機」，但更快速、更節省資源。

🐳 容器 vs 虛擬機

虛擬機 = 完整作業系統（笨重）

容器 = 只打包應用程式 + 依賴（輕量）

為什麼用 Docker？

  • 一致性： 「在我電腦上可以運行」不再是問題
  • 隔離性： 每個容器獨立，互不干擾
  • 可移植性： 一次打包，到處運行
  • 效率： 秒級啟動，資源占用少
  • 版本控制： 鏡像版本管理，隨時回滾

安裝 Docker

# macOS（使用 Homebrew） brew install --cask docker # Windows 下載 Docker Desktop: https://www.docker.com/products/docker-desktop # Ubuntu/Debian sudo apt update sudo apt install docker.io # 驗證安裝 docker --version docker run hello-world

💡 提示：初次運行需要啟動 Docker Desktop daemon。首次運行 docker run hello-world 會自動下載測試鏡像。

第二部分：核心概念

三個關鍵概念

概念 比喻 說明
鏡像（Image） 食譜 打包應用程式 + 環境的模板
容器（Container） 做好的菜 鏡像運行後的實例
Dockerfile 食譜寫法 定義如何建立鏡像的腳本

鏡像（Image）

鏡像是只讀模板，包含應用程式和所有依賴。類似「快照」或「模板」。

# 常見官方鏡像 ubuntu # Ubuntu 作業系統 nginx # Web 伺服器 python # Python 環境 node # Node.js 環境 mysql # 資料庫 redis # 快取資料庫

容器（Container）

容器是鏡像的運行實例。可以啟動、停止、刪除。一個鏡像可以運行多個容器。

🔄 鏡像 vs 容器關係

鏡像（Image）→ docker run → 容器（Container）

就像「類（Class）」和「物件（Object）」的關係

倉庫（Registry）

儲存和分享鏡像的地方。Docker Hub 是最大的公共倉庫。

  • Docker Hub： hub.docker.com（公共）
  • 私有倉庫： 公司內部搭建
  • Google Container Registry： gcr.io
  • GitHub Container Registry： ghcr.io

第三部分：常用命令

鏡像管理

# 搜尋鏡像 docker search python # 下載鏡像 docker pull python:3.11 docker pull nginx:latest # 查看本地鏡像 docker images # 刪除鏡像 docker rmi <image_id> docker image prune # 清理懸空鏡像 # 查看鏡像歷史 docker history <image_name>

容器管理

# 運行容器 docker run -d --name my-nginx -p 8080:80 nginx # 查看運行中的容器 docker ps # 查看所有容器（含停止的） docker ps -a # 停止容器 docker stop <container_id> # 啟動容器 docker start <container_id> # 刪除容器 docker rm <container_id> docker container prune # 清理停止的容器 # 進入容器互動 docker exec -it <container_id> bash # 查看容器日誌 docker logs <container_id>

常用選項解釋

docker run [選項] 鏡像 [命令] -d # 背景運行（detached mode） --name # 指定容器名稱 -p # 端口映射（host:container） -v # volumes 掛載（host:container） -e # 環境變數 --rm # 運行完自動刪除 -it # 互動模式（-i 互動，-t 終端）

實戰範例

# 運行 Python 容器並進入互動模式 docker run -it python:3.11 python # 運行 Nginx Web 伺服器 docker run -d --name web-server -p 8080:80 nginx # 運行 MySQL 資料庫 docker run -d --name mysql-db \ -e MYSQL_ROOT_PASSWORD=mysecret \ -p 3306:3306 \ mysql:8.0 # 掛載本地目錄到容器 docker run -d --name dev-env \ -v $(pwd):/app \ -w /app \ python:3.11 tail -f /dev/null

第四部分：Dockerfile 編寫

📝 Dockerfile 是文字檔案，定義如何建立鏡像。每一行是一個指令。

基本指令

FROM # 基礎鏡像 WORKDIR # 工作目錄 COPY # 複製檔案 RUN # 執行命令 EXPOSE # 暴露端口 CMD # 容器啟動命令 ENTRYPOINT # 容器啟動命令（不覆蓋） ENV # 環境變數

Python 應用 Dockerfile 範例

# 基礎鏡像 FROM python:3.11-slim # 設定工作目錄 WORKDIR /app # 複製依賴檔案 COPY requirements.txt . # 安裝依賴 RUN pip install --no-cache-dir -r requirements.txt # 複製應用程式 COPY . . # 暴露端口 EXPOSE 8000 # 啟動命令 CMD ["python", "app.py"]

建立和運行

# 建立鏡像 docker build -t my-python-app . # 運行容器 docker run -d --name my-app -p 8000:8000 my-python-app # 查看鏡像 docker images | grep my-python-app

Node.js 應用 Dockerfile 範例

FROM node:18-alpine WORKDIR /app COPY package*.json ./ RUN npm install COPY . . EXPOSE 3000 CMD ["npm", "start"]

💡 最佳實踐：使用 -slim 或 -alpine 版本，鏡像更小更安全。

第五部分：Docker Compose

🎼 Docker Compose 用於定義和運行多容器應用。用 YAML 檔案配置所有服務。

安裝 Compose

# 驗證安裝 docker-compose --version # macOS（Docker Desktop 已包含） # Ubuntu sudo apt install docker-compose

docker-compose.yml 範例

version: '3.8' services: # Web 應用 web: build: . ports: - "3000:3000" volumes: - .:/app depends_on: - db environment: - DATABASE_URL=mysql://root:secret@db:3306/mydb # 資料庫 db: image: mysql:8.0 environment: MYSQL_ROOT_PASSWORD: secret MYSQL_DATABASE: mydb volumes: - db-data:/var/lib/mysql # Redis 快取 redis: image: redis:alpine ports: - "6379:6379" volumes: db-data:

常用 Compose 命令

# 啟動所有服務 docker-compose up -d # 停止所有服務 docker-compose down # 查看服務狀態 docker-compose ps # 查看日誌 docker-compose logs -f # 重新建立鏡像 docker-compose up -d --build # 進入特定服務容器 docker-compose exec web bash

第六部分：實戰範例

範例 1：部署 WordPress

version: '3.8' services: wordpress: image: wordpress:latest ports: - "8080:80" environment: WORDPRESS_DB_HOST: db WORDPRESS_DB_USER: wordpress WORDPRESS_DB_PASSWORD: wppass WORDPRESS_DB_NAME: wordpress volumes: - wp-content:/var/www/html/wp-content db: image: mysql:8.0 environment: MYSQL_DATABASE: wordpress MYSQL_USER: wordpress MYSQL_PASSWORD: wppass MYSQL_ROOT_PASSWORD: rootpass volumes: - db-data:/var/lib/mysql volumes: wp-content: db-data:

運行：docker-compose up -d，訪問 http://localhost:8080

範例 2：Python Flask API

# app.py from flask import Flask app = Flask(__name__) @app.route('/') def home(): return "Hello from Docker!" if __name__ == '__main__': app.run(host='0.0.0.0', port=5000)
# Dockerfile FROM python:3.11-slim WORKDIR /app COPY requirements.txt . RUN pip install flask COPY . . EXPOSE 5000 CMD ["python", "app.py"]
# requirements.txt flask==2.3.0

建立並運行：docker build -t flask-app . && docker run -d -p 5000:5000 flask-app

範例 3：全棧應用（React + Node + MongoDB）

version: '3.8' services: # 前端 frontend: build: ./client ports: - "3000:3000" volumes: - ./client:/app - /app/node_modules environment: - CHOKIDAR_USEPOLLING=true # 後端 backend: build: ./server ports: - "5000:5000" volumes: - ./server:/app - /app/node_modules environment: - MONGODB_URI=mongodb://mongo:27017/myapp - PORT=5000 depends_on: - mongo # 資料庫 mongo: image: mongo:6 ports: - "27017:27017" volumes: - mongo-data:/data/db volumes: mongo-data:

範例 4：開發環境容器化

# docker-compose.dev.yml version: '3.8' services: dev-env: image: node:18-alpine container_name: my-dev-env working_dir: /app volumes: - .:/app - node_modules:/app/node_modules ports: - "3000:3000" command: tail -f /dev/null environment: - NODE_ENV=development volumes: node_modules:

運行：docker-compose -f docker-compose.dev.yml up -d

進入容器：docker exec -it my-dev-env sh

第七部分：最佳實踐

✅ 遵循這些最佳實踐，讓你的 Docker 更高效、更安全。

鏡像優化

  • 使用小鏡像： Alpine 或 slim 版本
  • 多階段建構： 減少最終鏡像大小
  • .dockerignore： 排除不必要的檔案
  • 層級優化： 把不常變化的指令放前面
# 多階段建構範例 # 第一階段：建構 FROM node:18 AS builder WORKDIR /app COPY . . RUN npm install && npm run build # 第二階段：運行 FROM nginx:alpine COPY --from=builder /app/dist /usr/share/nginx/html EXPOSE 80

安全性

  • 不要以 root 運行： 創建普通用戶
  • 掃描漏洞： docker scan <image>
  • 最小權限： 只暴露必要的端口
  • 更新鏡像： 定期更新基礎鏡像
# 創建非 root 用戶 FROM python:3.11-slim RUN useradd -m appuser WORKDIR /app COPY --chown=appuser:appuser . . USER appuser

日誌管理

  • 輸出到 stdout/stderr： Docker 自動收集
  • 使用日誌驅動： json-file, syslog, journald
  • 限制日誌大小： 避免磁碟爆滿

資源限制

# 限制 CPU 和記憶體 docker run -d --name limited-app \ --cpus="1.0" \ --memory="512m" \ my-app

版本控制

  • 固定版本： 不要用 latest，用具體版本號
  • 標籤策略： major.minor.patch
  • 鏡像倉庫： 使用私有倉庫儲存生產鏡像

⚠️ 常見錯誤：在容器中儲存持久化數據！使用 volumes 或 bind mounts 保存重要數據。
