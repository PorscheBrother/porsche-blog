LangGraph 深度解析：構建具有狀態與循環的 AI Agent 工作流

由 Porsche哥 編寫 • 2026年4月28日

從 Linear Chain 到 Cyclic Graph：AI 的進化

如果你使用過 LangChain，你一定熟悉所謂的「鏈 (Chain)」。鏈的邏輯是線性的：輸入 $\rightarrow$ 步驟 A $\rightarrow$ 步驟 B $\rightarrow$ 輸出。這在簡單的 RAG 或翻譯任務中表現良好。

但在面對複雜的 Agentic Workflows (代理工作流) 時，線性結構就失效了。真正的智能需要「循環 (Loops)」—— AI 執行一個動作，觀察結果，發現錯誤，然後回到上一步重新修正。這就是 LangGraph 存在的意義。

LangGraph 將 LLM 應用視為一個 有向圖 (Directed Graph)，讓你可以精確定義狀態的流轉與反饋循環，將 AI 從簡單的對話框轉化為能夠自我修正的自動化系統。

LangGraph 核心概念：

  • State (狀態)： 全局共享的記憶體，記錄目前進度與數據 (例如：已搜尋的關鍵字、目前的草稿)
  • Nodes (節點)： 具備邏輯的函數。接收 State $\rightarrow$ 執行動作 $\rightarrow$ 更新 State
  • Edges (邊)： 定義節點之間的轉移路徑
  • Conditional Edges (條件邊)： 根據 LLM 的判斷決定下一步去哪裡 (例如：如果結果不滿意 $\rightarrow$ 回到搜尋節點)

LangGraph 實戰教學：構建一個「自我修正」的寫作助理

假設我們要構建一個助理，流程是：搜尋資訊 $\rightarrow$ 撰寫草稿 $\rightarrow$ 審查品質 $\rightarrow$ (若不合格) $\rightarrow$ 重新撰寫。

第一步：定義 State

State 是 Graph 的靈魂。它定義了所有節點之間傳遞的數據結構。

from typing import TypedDict, List class AgentState(TypedDict): "messages": List[str] "draft": str "critique": str "iterations": int

第二步：定義 Nodes (邏輯單元)

每個節點就是一個簡單的 Python 函數，負責處理狀態。

def writer_node(state: AgentState): // 使用 LLM 根據訊息撰寫草稿 draft = llm.invoke("請撰寫關於...的文章") return {"draft": draft, "iterations": state["iterations"] + 1} def critic_node(state: AgentState): // 使用 LLM 審查草稿 critique = llm.invoke("請批評這篇草稿：" + state["draft"]) return {"critique": critique}

第三步：定義圖結構 (Graph)

將節點連接起來，並設定條件跳轉。

from langgraph.graph import StateGraph, END workflow = StateGraph(AgentState) workflow.add_node("writer", writer_node) workflow.add_node("critic", critic_node) workflow.set_entry_point("writer") workflow.add_edge("writer", "critic") // 條件邊：判斷是否需要重新撰寫 def should_continue(state: AgentState): if "合格" in state["critique"] or state["iterations"] > 3: return END return "writer" workflow.add_conditional_edges("critic", should_continue) app = workflow.compile()

LangGraph 的核心應用場景

當你需要 AI 具備「思考 $\rightarrow$ 行動 $\rightarrow$ 反思」的閉環能力時，LangGraph 是首選：

  • 自動化程式碼開發 (Coding Agents)： 寫代碼 $\rightarrow$ 運行測試 $\rightarrow$ 根據錯誤訊息修改代碼 $\rightarrow$ 再次測試。這是目前最頂尖的 AI 寫作工作流。
  • 複雜的客戶支持系統： 根據用戶意圖跳轉到不同專業節點 $\rightarrow$ 獲取外部 API 數據 $\rightarrow$ 檢查答案是否滿足用戶 $\rightarrow$ 回覆。
  • 多代理協作 (Multi-Agent Collaboration)： 定義多個節點（如：產品經理、工程師、測試員），讓他們在圖中流轉，互相審核對方的產出。
  • 研究報告生成： 規劃大綱 $\rightarrow$ 分段搜尋 $\rightarrow$ 整合內容 $\rightarrow$ 檢查邏輯漏洞 $\rightarrow$ 補全缺失資訊。

總結：掌控 AI 的邏輯流

LangGraph 的本質是將 控制權 從 LLM 的隨機生成，交還給開發者。我們不再只是祈禱 LLM 能一次給出正確答案，而是通過構建嚴謹的狀態圖，強迫 AI 在正確的邏輯路徑上運行。

如果你想讓你的 AI 助理從「會聊天」變成「能工作」，學習 LangGraph 是必經之路。記住：最強大的 AI 不是最強的模型，而是最精巧的工作流。直接、高效、有邏輯，這才是 Porsche哥 的開發之道。
