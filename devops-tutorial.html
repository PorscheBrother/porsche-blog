DevOps 完整教學

由 Porsche哥 編寫 • 2026年4月6日

什麼是 DevOps？

DevOps 是一種文化哲學、實踐和工具的結合，旨在提高組織交付應用程式和服務的能力，使其能夠以更高的速度演進和改進產品，比使用傳統軟體開發和基礎設施管理流程更快。

DevOps 核心價值：

  • 合作： 打破開發和運維之間的隔閡
  • 自動化： 自動化重複性任務以提高效率
  • 持續改進： 持續監控、測量和優化
  • 以客戶為中心： 快速響應客戶需求和市場變化

DevOps 的演變

DevOps 沒有從虛無中誕生，而是從軟體開發演化過程中自然產生的：

  • 瀑布模型： 順序性開發，變更緩慢
  • 敏捷開發： 迭代開發，但開發和運維仍然分離
  • DevOps： 開發、測試、運維全程合作與自動化
  • Site Reliability Engineering (SRE)： Google 開發的，將軟體工程應用於基礎設施問題

DevOps 核心概念

1. 持續整合與持續交付 (CI/CD)

CI/CD 是 DevOps 的核心實踐，通過自動化構建、測試和部署流程來快速、可靠地交付軟體變更。

# 基本 CI/CD 流程示例（使用 GitHub Actions） name: CI/CD Pipeline on: push: branches: [ main ] pull_request: branches: [ main ] jobs: build: runs-on: ubuntu-latest steps: - uses: actions/checkout@v2 - name: Set up Python uses: actions/setup-python@v2 with: python-version: '3.8' - name: Install dependencies run: | python -m pip install --upgrade pip pip install flake8 pytest if [ -f requirements.txt ]; then pip install -r requirements.txt; fi - name: Lint with flake8 run: | # 斷開執行以便在失敗時顯示所有錯誤 flake8 . --count --select=E9,F63,F7,F82 --show-source --statistics flake8 . --count --exit-zero --max-complexity=10 --max-line-length=127 --statistics - name: Test with pytest run: | pytest - name: Build and push Docker image run: | docker build . -t myapp:${{ github.sha }} echo "Image built"

2. 基礎設施即代碼 (Infrastructure as Code, IaC)

透過程式碼來管理和佈署基礎設施，而不是手動配置硬體和作業系統。

# Terraform 示例：建立 AWS EC2 執行個體 provider "aws" { region = "us-west-2" } resource "aws_instance" "web" { ami = "ami-0c55b159cbfafe1f0" instance_type = "t2.micro" tags = { Name = "hello-world" } }
# AWS CloudFormation 示例（YAML） Resources: MyEC2Instance: Type: "AWS::EC2::Instance" Properties: ImageId: "ami-0ff8a91507f77f867" InstanceType: t2.micro Tags: - Key: "Name" Value: "MyInstance"

3. 監控與日誌

有效的監控和日誌記錄是理解系統行為、故障排除和效能優化的關鍵。

監控三大支柱：

  • 指標 (Metrics)： 數值型資料，如 CPU 使用率、記憶體消耗
  • 日誌 (Logs)： 離散事件記錄，如錯誤訊息、使用者操作
  • 追蹤 (Traces)： 分佈式系統中請求的路徑資訊
# Prometheus 指標範例 # HELP http_requests_total The total number of HTTP requests. # TYPE http_requests_total counter http_requests_total{method="post",code="200"} 1027 http_requests_total{method="get",code="200"} 2338

4. 微服務與容器化

DevOps 與微服務架構和容器技術密切相關，因為它們共同啟用了快速的開發和部署週期。

# Dockerfile 示例 FROM node:14 WORKDIR /app COPY package*.json ./ RUN npm install COPY . . EXPOSE 3000 CMD [ "node", "server.js" ]
# docker-compose.yml 示例 version: '3' services: web: build: . ports: - "5000:5000" redis: image: "redis:alpine"

DevOps 工具鏈

常見 DevOps 工具按功能分類：

# 版本控制 Git, SVN, Mercurial # 持續整合 Jenkins, GitLab CI, GitHub Actions, CircleCI, Travis CI # 配置管理 Ansible, Chef, Puppet, SaltStack # 基礎設施即代碼 Terraform, AWS CloudFormation, Azure Resource Manager, Google Deployment Manager # 容器 orchestration Kubernetes, Docker Swarm, Apache Mesos # 監控與日誌 Prometheus, Grafana, ELK Stack (Elasticsearch, Logstash, Kibana), Datadog, New Relic # 協作與專案管理 Jira, Confluence, Slack, Microsoft Teams

DevOps 最佳實踐

成功實施 DevOps 的關鍵實踐：

# 1. 從小處著手，但要思考宏大 # 從一個小專案或服務開始實踐 DevOps 原則 # 證明價值後再擴展到整個組織 # 2. 自動化一切可自動化的事情 # 建置、測試、部署、基礎設施佈署、監控告警 # 目標是讓人類專注於創造性工作而非重複性任務 # 3. 實施持續反饋循環 # 從監控中獲得見解並快速回饋給開發團隊 # 使用蓝绿部署或金丝雀发布降低風险 # 4. 培養協作文化 # 打破 silos，鼓勵跨職能團隊合作 # 共享目標和責任，而非各自為政 # 5. 以度量驅動改進 # 定義關鍵績效指標 (KPIs) 並持續追蹤 # 例如：部署頻率、變更前置時間、平均復原時間、變更失敗率

DevOps 與雲端運算

雲端運算為 DevOps 提供了理想的平台，因為它提供了：

  • 即時資源佈署： 透過 API 快速取得計算資源
  • 彈性伸縮： 根據負載自動調整資源規模
  • 管理服務： 減輕運維負擔，專注於業務邏輯
  • 全球佈署： 易於在多個地理位置佈署應用

學習路線與資源

🎯 DevOps 學習建議路線：

# 第一階段：基礎知識 # 學習 Linux 基礎、網路概念、腳本語言 (Bash/Python) # 理解軟體開發生命週期 (SDLC) # 第二階段：工具熟練 # 精通一個版本控制系統 (Git) # 學習一個 CI/CD 工具 (Jenkins 或 GitHub Actions) # 熟悉一個配置管理工具 (Ansible) # 初探容器化 (Docker) 和 orchestration (Kubernetes) # 第三階段：實踐整合 # 建立完整的 CI/CD pipeline # 使用 IaC 管理基礎設施 # 實施監控和日誌解決方案 # 參與真實的生產環境維護 # 第四階段：專業深化 # 學習雲端平台 (AWS/Azure/GCP) # 深入微服務架構和服務網格 # 探索進階觀測性 (Observability) 技術 # 獲得相關認證 (例如：AWS DevOps Engineer, Docker Certified, CKA)

結語

DevOps 不僅是一套工具或一套流程，更重要的是一種文化和心態的轉變。它要求組織打破傳統的部門牆，培養共同負責產品全生命週期的思維方式。

隨著技術的快速演進，DevOps 將繼續發展和適應新的挑戰和機遇。無論您是開發人員、系統管理員，還是想要進入這個領域的新手，理解和實踐 DevOps 原則都將使您在當今的科技產業中具備競爭優勢。

記住：DevOps 的旅程是持續改進的旅程。從小處著手，持續學習，不斷迭代和優化您的實踐方式。

💡 關鍵提醒：DevOps 不是目的地，而是一段旅程。專注於文化變革和持續改進，而非只是追逐最新的工具或技術。
